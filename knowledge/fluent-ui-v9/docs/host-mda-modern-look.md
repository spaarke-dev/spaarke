# Modern, refreshed look for model-driven apps

Model-driven apps have updated styling — fonts, colors, borders, shadows — that align with the [Microsoft Fluent design system](https://react.fluentui.dev/). The updated look provides consistency, quality, and Microsoft-wide platform coherence, plus a foundation for extensibility and future dark mode support.

> **Important** — With the **2026 Wave 1** release, all users MUST use the **New Look**. Makers can't switch a model-driven app back to the classic look. This doesn't include the **Header and navigation refresh (preview)**. The New look toggle was removed for users with the **2025 Wave 1** release.

## What's included

- Updated styling in form, view, and dashboard pages — drop shadows, brighter backgrounds, "floating" appearance to visually separate sections.
- New **Fluent-based controls** in forms, business process flows, and dialogs. Dialogs resize height automatically based on content.
- **Power Apps grid** replaces the read-only grid in views and dashboards.
- New mechanism for customizing the **app header colors** to match personal or organizational branding. See [Use modern themes](https://learn.microsoft.com/power-apps/maker/model-driven-apps/modern-theme-overrides).

### Command bar

Floating command bar aligns with the Microsoft 365 experience — consistent spacing, rounded corners, elevation. Sits in a separate section at the top of the page.

### View pages

- New command bar + updated grid areas leveraging elevation.
- **Power Apps grid control** replaces the read-only grid. Features infinite scrolling. Also appears in subgrids + associated grids in main forms + dashboards.
- Inline editing supported via `Enable filtering` property.

### Form pages

- New command bar + refreshed headers, tabs, sections, business process flows.
- Quick views, card forms, headers, sitemaps, and timeline controls have updated styling.

### Field controls

- Text input, action input, lookup, and checkbox controls **built using Fluent components**.
- Field sections (containers for fields on a form) have streamlined design.
- **Icons now on the right side of field labels.** Some redundant icons removed.
- Input + error message styling refreshed based on Fluent design.

### Dashboard page

- System dashboard uses new command bar + section-style elevation.
- When modern experience is on (or monthly channel), dashboard grids use the new Power Apps grid control. Not on by default.

### Chart controls

Chart controls on view pages + dashboards use a new color palette. **Custom chart colors are overridden** unless you set:

```xml
<Chart CustomColorOverride="true">
```

## Header and navigation refresh (preview)

> Preview feature; subject to change.

Modern app header, streamlined sitemap, condensed page headers with compact commands.

**Enable for an existing app**:
1. In the app designer, **Settings** on command bar.
2. **Features** → enable **Header and navigation refresh**.

Automatically applied to all apps generated by Plan designer.

## Known limitations

- **Mobile + mail apps** don't support the modern look.
- **Theme switching / dark mode** isn't supported yet.
- **Classic theme customizations no longer honored.** Override the app header colors only.
- **Custom icons**: only SVG. PNG and other formats are ignored — default icon shown.

## FAQ — critical for Spaarke

### Can I use modern look in PCF / custom code components?

> "The modern theme in use is **passed to Power Apps component framework** components, so you can [style your components with modern theming](https://learn.microsoft.com/power-apps/developer/component-framework/fluent-modern-theming)."

✅ Spaarke PCFs that consume `context.fluentDesignLanguage.tokenTheme` will inherit modern theme automatically. See [`pcf-modern-theming.md`](./pcf-modern-theming.md).

### Can I use modern look in custom pages?

> "You can use modern controls with custom pages and the modern, refreshed look. **Currently, custom pages don't use the modern theme.**"

⚠️ **Critical**: Power Apps **custom pages** (canvas-style pages embedded in MDA) do NOT auto-receive the modern theme. This is distinct from **Spaarke's Code Pages** (React 18 SPAs delivered as Dataverse web resources) — see [`host-code-pages-styling.md`](./host-code-pages-styling.md) for Spaarke's approach.

### Can I change the app header color?

Yes — to match organization branding. Other theme customization options aren't available yet.

## Revert to classic look (admin only)

> With 2026 Wave 1, New Look is mandatory. End users can't switch. **Admins** can still turn off new look for all users via the `New look for model driven apps` app setting:

1. Open https://make.powerapps.com/
2. Open an existing solution with model-driven apps
3. **Add Existing** → **More** → **Setting**
4. Search **New look**
5. Select **New look for model driven apps** → **Add**
6. Update **Setting Environment Value** to **No**
7. Add the apps; update each app's **value** to **No**
8. Save + publish all customizations

## Spaarke implications

| Surface | Impact |
|---|---|
| **PCFs (Spaarke `Spaarke.UI.Components` + per-PCF)** | Auto-inherit modern theme via `context.fluentDesignLanguage.tokenTheme`. Use `appearance="filled-darker"` on Input/Combobox to match the "filled" look the field controls use. See [`community/birkelbach-style-fluent-ui-9-pcfs.md`](./community/birkelbach-style-fluent-ui-9-pcfs.md). |
| **Code Pages (Spaarke React 18 SPAs)** | Do NOT auto-inherit modern theme. Developer must pick a theme at root + match visual standards manually. See [`host-code-pages-styling.md`](./host-code-pages-styling.md). |
| **Form-side rendering** | Power Apps grid handles read-only and inline-edit grid behavior. Don't reinvent grid scrolling/edit semantics in PCFs. |
| **Icons** | If you ship an SVG icon in a PCF or sitemap, that's fine. PNGs in sitemap are ignored. |
| **Chart customization** | If a Spaarke chart uses brand colors, add `CustomColorOverride="true"` or those colors will be overridden by modern palette. |
