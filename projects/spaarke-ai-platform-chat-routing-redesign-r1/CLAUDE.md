## Key Technical Constraints

### ADRs (binding — load via `adr-check`)

- **ADR-001 Minimal API**: New endpoints `/api/ai/playbooks/by-code/{code}`, `/api/memory/promotions/{id}/{approve|reject}`
- **ADR-008 Endpoint filters**: Tenant scoping on new endpoints
- **ADR-010 DI minimalism**: No `IServiceProvider.GetService<T>()` in endpoint signatures; concrete-class Null-Object
- **ADR-013 AI facade boundary**: Memory tools cross `Services/Ai/PublicContracts/` cleanly. **NO** direct injection of `IOpenAiClient` / `IPlaybookService` into CRUD code.
- **ADR-014 AI caching**: 5-min TTL on `/by-code/` resolution; tenant-scoped
- **ADR-015 AI Data Governance** (CRITICAL — Tier 1 logging): NEVER log user message content, file content, recall results, memory facts. Memory event payloads carry deterministic IDs + 80-char `factSummary` preview only.
- **ADR-018 Typed options**: Fix `Workspace:SummarizePlaybookId` raw indexer; add `WorkspaceOptions.SummarizePlaybookCode`
- **ADR-019 ProblemDetails**: 404 shape on `/by-code/` not found
- **ADR-029 BFF Publish Hygiene** (CRITICAL): Per-task `dotnet publish` measurement; ≤60 MB ceiling; current baseline 45.65 MB. `<PublishTrimmed>` / `<PublishAot>` PROHIBITED.
- **ADR-030 v2 PaneEventBus** (amended 2026-06-21 by THIS project): 5-channel union now includes `memory`; `MemoryPaneEvent` has 5 initial discriminants. Memory-domain events dispatch via `memory` channel, NOT namespaced inside `workspace.*`.
- **ADR-032 BFF Null-Object Kill-Switch**: Any feature-gated memory tool registration uses P1/P2/P3; asymmetric registration anti-pattern BANNED
- **ADR-033 Streaming chat-tool side channel**: Path 3 streaming preservation via context-injected `DocumentStreamWriter`; new memory tools return single `ToolResult`

See `notes/lessons-learned.md` §3 + spec.md MVP Scope Cut section.

This finding shrinks the follow-on product project's scope substantially.

This is exactly the failure mode CLAUDE.md §10 F.1 warns against. Fix landed (commit `3119c11ef`), merged via PR #491. Worth a future health-check hook.
- **Daily-briefing-r4 task 031 (Path A.5 canonical pattern)** landed in master DURING this project — its Path A.5 thin-dispatch pattern (`IConsumerRoutingService` + `IInvokePlaybookAi`) became the canonical reference for all future endpoints. Documented in `docs/architecture/ai-architecture-playbook-consumer-routing.md` (renamed + augmented at task 150).
