## Problem Statement

The SmartToDo Kanban board and TodoDetailSidePane are two separate surfaces communicating via BroadcastChannel across iframes. This causes:

- Duplicate MSAL auth sessions
- Duplicate theme detection
- Full refetch on every save (instead of single-item update)
- Fixed 400px side pane (not resizable)
- 6-layer navigation detection hack for auto-close (~140 lines)
- Two deployment artifacts

## Solution Summary

Merge both into a single unified Code Page (`sprk_smarttodo`) with:

- Inline resizable detail panel (PanelSplitter from shared library)
- TodoContext for shared state (items, selectedEventId, optimistic updates)
- Single React 19 tree — no iframe, no BroadcastChannel
- Panel width persisted via localStorage
- One deployment artifact
