## Design Decisions

1. **New `useTwoPanelLayout`** hook (not generalizing three-panel) — cleaner API, no AnalysisWorkspace regression
2. **Copy Kanban components** from LegalWorkspace (not extract to shared lib) — keeps LegalWorkspace stable
3. **TodoDetail callback-based API** — accepts onSearchContacts, onSave callbacks; decouples from Xrm
4. **localStorage** for panel width/visibility persistence (cross-session)
5. **TodoContext** — React context for shared state between Kanban and Detail panel

## Constraints

- MUST use React 19 `createRoot()` (Code Page)
- MUST use Fluent v9 FluentProvider with theme detection
- MUST use Vite + vite-plugin-singlefile for single HTML output
- MUST NOT use BroadcastChannel for Kanban-to-Detail communication
- MUST NOT use Xrm.App.sidePanes for detail panel in Kanban context
