# D-01: IChatClient streaming mocking — BUILD LOCAL (no Microsoft-shipped helper exists)

**Status**: Locked (2026-05-31, Phase 0 task 003 verdict)
**Source**: [`design.md`](../design.md) §5.1; FR-02
**Binding on**: Phase 1 task P1.B1 (the `AsyncEnumerableHelpers` task — to be generated), and any subsequent task that mocks `IChatClient.GetStreamingResponseAsync` consumption

---

## Context

[`design.md`](../design.md) §5.1 left open whether the project's `IAsyncEnumerable<ChatResponseUpdate>` test helper should consume a Microsoft-shipped helper (preferred per owner direction "use the most robust available") or be hand-rolled. The currently-referenced `Microsoft.Extensions.AI` package is at v10.3.0 (`tests/unit/Sprk.Bff.Api.Tests/Sprk.Bff.Api.Tests.csproj` line 43; `src/server/api/Sprk.Bff.Api/Sprk.Bff.Api.csproj` line 31). This verdict resolves §5.1 so Phase 1 task P1.B1 can be authored against a concrete plan.

## Verdict

**BUILD LOCAL.** No Microsoft-shipped `Microsoft.Extensions.AI.Testing` (or equivalent companion) NuGet package exists that publishes a public `IChatClient` streaming mock. The relevant Microsoft test helper, `TestChatClient`, lives ONLY in the dotnet/extensions repo at `test/Libraries/Microsoft.Extensions.AI.Abstractions.Tests/TestChatClient.cs` — i.e., inside the SDK's internal test fixtures, NOT redistributed to NuGet. Consuming it would require either copy-pasting MIT-licensed source into our `Mocks/` folder or vendoring it via a custom mechanism. The mechanically equivalent decision is to author a small local helper that follows the same pattern (a callback-driven `IChatClient` stub plus an `IAsyncEnumerable<ChatResponseUpdate>` helper).

## Implication for P1.B1

Phase 1 task P1.B1 (`AsyncEnumerableHelpers` — to be generated by `task-create`) should:

1. **Author** `tests/unit/Sprk.Bff.Api.Tests/Mocks/AsyncEnumerableHelpers.cs` — a small static helper exposing `IAsyncEnumerable<ChatResponseUpdate> FromChunks(params string[] chunks)` plus `FromChunks(IEnumerable<ChatResponseUpdate>)` overloads, and (optionally) a thin `FakeChatClient` companion that follows the dotnet/extensions `TestChatClient` callback-property pattern (`GetResponseAsyncCallback`, `GetStreamingResponseAsyncCallback`).
2. **Mirror naming/shape** to the Microsoft `TestChatClient` pattern (callback properties, not virtual methods) so a future swap to a Microsoft-shipped helper is mechanical if one ships later.
3. **NOT** add a `Microsoft.Extensions.AI.Testing` `<PackageReference>` to the test csproj — the package does not exist on NuGet.org as of 2026-05-31.
4. **MAY** cite this decision file + the `TestChatClient.cs` URL in code comments at the top of `AsyncEnumerableHelpers.cs` so future maintainers see the Microsoft-pattern lineage.
5. **MUST** treat the helper as test-only infrastructure (no production-code coupling) per NFR-01.

The §5.1 effort estimate ("~4-6h hand-rolled") stands; the saved ~4-6h of "use Microsoft helper" was contingent on a NuGet package that doesn't exist.
