## Parallel-Safety Verification Rule

**Pre-flight check before every wave dispatch**:

A task is `parallel-safe: true` if and only if its `<relevant-files>` list has zero intersection with the `<relevant-files>` of every other `parallel-safe: true` task in the same wave. The orchestrator MUST verify this before dispatching agents in parallel. If any intersection is found, the wave is split or downgraded.

For each wave above, the orchestrator should:
1. Read each task's `<relevant-files>` block
2. Build the union of touched-files across the wave
3. Confirm zero duplicates (each file appears in at most one task's list)
4. IF duplicates → split wave OR demote conflicting tasks to next wave
