What's actually missing today
A broken test suite means we have these verification layers, but not all of them:

Layer	What it catches	What it MISSES
✅ Build verification (dotnet build)	Compile errors, type mismatches, signature breakage	Runtime behavior, business logic
✅ Capture-BffBaseline.ps1 synthetic smoke	Route registration, startup health, auth pipeline, gross 500s on 323 routes	Edge cases, business logic correctness, error paths
✅ Hash-verify + healthz (Deploy-BffApi.ps1)	Mechanical deploy success	Anything functional
✅ User E2E testing	What the human user notices	Background jobs, rare paths, exception handling
✅ App Insights observation	Production-rate exceptions, latency	Pre-prod regressions; takes days to surface
❌ Unit tests	Business logic correctness, edge cases, regression prevention	—
❌ Integration tests	What happens when ServiceBus throws / Cosmos rate-limits / Dataverse returns 403	—
The bottom two are non-functional today. Every change to BFF business logic ships without unit-test verification.

How this project survived without working tests
The Phase 4 facade migration shipped successfully without tests, but partly by luck:

It was a structural refactor (interface delegation, namespace moves) — no behavior change
The synthetic smoke caught structural breaks (routes still register, auth still works)
User E2E caught the one real bug (/api prefix in LegalWorkspace)
Code review caught the 3 important findings
But the next BFF change is likely business logic — edge cases in playbook resolution, error handling in invoice extraction, retry behavior in webhooks. Synthetic smoke + E2E won't catch those.

What the gap actually costs
Cost	Frequency	Severity
Can't ask "did you add a test?" in PR review	Every PR	Low (most PRs skip this anyway)
Can't gate CI on dotnet test (283 failures = noise)	Every PR	High — CI provides no functional signal

Test debt compounds (more drift, more lost context)	Continuous	Medium — gets harder the longer you wait
Refactor confidence is artificially low → smaller, more cautious changes	Continuous	Medium — slows velocity
Is it urgent?
Not blocking anything today. The other verification layers caught everything this project needed. But it's accumulating compound interest:

Today: 283 failures, ~13-20h to repair to "≤30 remaining"
In 3 months: more drift from in-flight projects, probably 350-400 failures, 20-30h to repair
In 6 months: significant chunks may be unrepairable without rewriting the tests from scratch
Eventually: the test suite becomes vestigial, gets deleted, BFF has no unit-test layer at all
So: not an emergency, but it should be the next BFF project rather than the 5th-next.
