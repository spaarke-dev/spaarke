## 5. Process Rules

### Sub-agent write boundary (per root [CLAUDE.md](../../CLAUDE.md) §3)

Sub-agents launched via the Agent tool CANNOT write to `.claude/` paths. Tasks 080 (`.claude/constraints/bff-extensions.md`) and 083 (root `CLAUDE.md`) are `parallel-safe: false` AND `main-session-only: true`. The main session edits these sequentially in parallel-time with the agent-dispatched wave.

### Repair-not-rewrite (NFR-09)

Every task POML's metadata declares `<repair-not-rewrite>true</repair-not-rewrite>`. `task-execute` verifies this before starting work. A diff replacing >50% of a test file's lines triggers §4.8 escalation — agent halts, files `escalations/rewrite-request-T-XX-FileName.md`, waits for owner approval. Hard limit: 5% of touched files; if exceeded, project pauses for design-review.

### Anti-parallelism guard (NFR-07)

Tasks that modify `CustomWebAppFactory.cs` (018, 019) run in **isolation** — never concurrent with other repair tasks. Factory changes have global blast radius across 4,844 baseline tests.

### Cross-track regression (FR-21)

Full-suite check runs between every Phase 2+3 wave (operational cadence — pipeline enforces this as the equivalent of design.md's "every 4 hours"). Any new failure in tests not currently being worked HALTS all tracks until investigated.

### Build verification between waves

`dotnet build src/server/api/Sprk.Bff.Api/` + `dotnet build tests/unit/Sprk.Bff.Api.Tests/` after every wave that modified `.cs` files. Catches incoherent parallel-agent changes before they compound.

### Archive method (NFR-06)

Tests are archived by **rename only**: `XxxTests.cs` → `XxxTests.cs.archived-YYYY-MM-DD`. Delete-from-disk is prohibited (preserves git history continuity and restore option).
