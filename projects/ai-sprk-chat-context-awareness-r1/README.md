## Overview

Replace SprkChat's hardcoded playbook mapping with a data-driven system. A new Dataverse table (`sprk_aichatcontextmap`) lets admins configure which playbooks appear in which context (entity type + page type), with zero code deployment required to change mappings.

## Problem Statement

SprkChat opens with a hardcoded `DEFAULT_PLAYBOOK_MAP` regardless of page context. Admins cannot change playbook-to-context mappings without code deployment. The AI has no awareness of what entity/page the user is viewing.

## Solution Summary

1. **Data-Driven Mapping** — `sprk_aichatcontextmap` Dataverse table + BFF endpoint + Redis caching
2. **Multi-Playbook UX** — Dynamic playbook selector populated from context-mapping response
3. **Entity Metadata Enrichment** — System prompt enriched with entity name + page type
4. **Admin Experience** — Model-driven form + cache refresh button

## Graduation Criteria

- [ ] Admin creates mapping record → SprkChat picks up new playbook (no deploy)
- [ ] SprkChat auto-selects correct playbook per entity/page context
- [ ] `GET /api/ai/chat/context-mappings` ≤50ms p95 (Redis hit)
- [ ] Pane opens ≤300ms p95
- [ ] Generic chat mode when no mappings exist
- [ ] Seed data importable via PAC CLI
- [ ] AI response references entity name on matter form (Phase 3)
- [ ] Entity name absent from structured Application Insights logs
- [ ] Existing SSE streaming not regressed
- [ ] Backward-compatible ChatHostContext deserialization

## Scope

### In Scope
- Phase 1: Data-driven mapping (Dataverse entity, BFF service, client integration)
- Phase 2: Multi-playbook UX (selector, switching, sessionStorage cache)
- Phase 3: Entity metadata enrichment (system prompt, token budget guard)
- Phase 4: Admin experience (model-driven form, cache refresh)

### Out of Scope
- Slash command menu / quick-action bar (separate project)
- Changes to `sprk_aiplaybook` table structure
- Multi-tenant mapping isolation
- AI pipeline/middleware/tool registration changes

## Key Decisions

| Decision | Resolution |
|----------|-----------|
| No-mapping fallback | Hide selector, show generic chat (null PlaybookId) |
| Cache TTL | 30-min Redis; Phase 4 adds manual refresh |
| Workspace detection | Explicit allowlist (not broad `sprk_` match) |
| PlaybookId nullability | `Guid?` on ChatSession + CreateSessionAsync |

## Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| Enrichment disrupts playbook prompts | High | Strictly appended last; never mid-prompt |
| Entity name PII in logs | High | Never in structured properties; Debug-level only |
| Token budget exceeded | Medium | Check budget before appending; skip + log warning |
| PlaybookChatContextProvider untested | Medium | Write unit tests before modifying |
