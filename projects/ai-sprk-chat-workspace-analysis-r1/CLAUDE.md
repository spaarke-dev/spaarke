## Project Context

Replace SprkChat's hardcoded `DEFAULT_PLAYBOOK_MAP` with data-driven context mapping via `sprk_aichatcontextmap` Dataverse table. 4 phases: MVP mapping → multi-playbook UX → entity enrichment → admin experience.

## Key Constraints (MUST Rules)

- MUST call `.AddAiAuthorizationFilter()` on context-mappings endpoint
- MUST call `ExtractTenantId()` and return 400 if null
- MUST check Redis before querying Dataverse
- MUST use `string? PageType = null` default on ChatHostContext
- MUST append Phase 3 enrichment AFTER playbook system prompt
- MUST guard enrichment: skip if EntityName/PageType null
- MUST cap enrichment block at ≤100 tokens
- MUST NOT log EntityName in structured telemetry
- MUST NOT block pane open on cold Dataverse query
- MUST NOT introduce REST endpoints returning AI content
- MUST NOT use `sprk_` wildcard for workspace detection — use allowlist
