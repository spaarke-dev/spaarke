## Reframing

Task POML title said "Migrate ESLint v8 → v9 (flat config)". In reality:

- `package.json` already pinned `eslint@^9.17.0`.
- No `.eslintrc.json`, `.eslintrc.js`, or `eslint.config.*` existed at the package root.
- The lint script (`"eslint src --ext .ts,.tsx"`) used the legacy `--ext` flag and was failing with
  `ESLint couldn't find an eslint.config.(js|mjs|cjs) file`.

So the work was **create the missing flat config + align the lint script + register the
minimum useful plugin set**, not a migration. Documented in the eslint.config.js header.

## File-scope decision: `.js` files excluded

The package historically contained two `.js` files under `src/`:

- `src/__mocks__/diff.js` — CommonJS mock for Jest (uses `module.exports`)
- `src/services/EntityCreationService.js` — appears to be a compiled/legacy duplicate of the `.ts` version (uses `console`, has pre-existing `// eslint-disable` directives)

Including these in lint scope under ESLint v9 flat-config required either:
1. Adding Node/CommonJS globals to an additional override block, OR
2. Excluding `.js` from the lint scope

Per task constraint "Do NOT modify code", and because both files are non-TypeScript and
non-production-loadbearing (one is a Jest mock, the other is a stale duplicate likely to be
deleted in a future cleanup), `eslint.config.js` excludes `**/*.js`, `**/*.cjs`, `**/*.mjs`.

**Carry-over**: B-11 or a later cleanup task should evaluate whether
`src/services/EntityCreationService.js` is still consumed; if not, delete it.

### B-11 / future-task escalation list

The following warnings represent **real correctness issues** and should be addressed BEFORE the warnings become errors again (i.e., before promoting the rules back to `error`):

1. **`react-hooks/rules-of-hooks` × 10** in `DatasetGrid/GridView.tsx` and `DatasetGrid/ListView.tsx`.
   Pattern: early `if (virtualization.shouldVirtualize && props.records.length > 1000) return (...)`
   at line 90 (GridView) is followed by `useMemo`/`useCallback` calls. The fix is to hoist all
   hooks above the conditional `return`, or invert the structure so the hooks always run.
   **Owner**: a dedicated DatasetGrid refactor task (not in current R4 backlog; candidate for R5).

2. **`react-hooks/exhaustive-deps` × 22** — same files plus others. Real missing deps in some
   cases; needs case-by-case review. Lower priority than rules-of-hooks.
