## Overview

Extend the existing Playbook execution engine with a `CreateNotification` node executor and a `PlaybookSchedulerService` to generate `appnotification` records on schedule. Build a Daily Digest Code Page that queries those notifications, groups by category, and presents a narrative TL;DR with action links and optional AI-generated briefing summary.

## Problem Statement

Users have no consolidated view of what happened across their matters, projects, tasks, and documents. Notifications are scattered — the MDA bell icon shows individual items but lacks categorization, narrative summaries, or configurable channels. There is no scheduled notification generation from playbook-driven rules.

## Proposed Solution

1. **CreateNotificationNodeExecutor** (ActionType 50) — new playbook node that creates `appnotification` records via Dataverse Web API
2. **PlaybookSchedulerService** — BackgroundService that runs notification-mode playbooks on schedule
3. **NotificationService** — singleton for inline notification creation from BFF endpoints
4. **Daily Digest Code Page** (`sprk_dailyupdate`) — React 19 dialog showing categorized notification summary with AI briefing
5. **7 notification playbooks** — deterministic rules for tasks, documents, emails, events, activity, and work assignments

## Scope

### In Scope
- `CreateNotificationNodeExecutor` (ActionType 50) following existing `NodeExecutorRegistry` pattern
- `createNotification` canvas type in PlaybookBuilder node palette
- `PlaybookSchedulerService` BackgroundService for scheduled notification playbooks
- `PlaybookRunContext` extension with `UserId` and `UserPreferences`
- `NotificationService` singleton for inline notifications from BFF endpoints
- Inline notification calls in document upload, analysis complete, email received, work assignment
- Notification deduplication (idempotency check)
- 7 notification playbooks (`sprk_playbooktype = Notification (2)`)
- `sprk_dailyupdate` Code Page (React 19, Vite)
- AI briefing summary endpoint (`POST /api/ai/daily-briefing/summarize`)
- User-customizable channel preferences (opt-out model)
- Clear/dismiss items, mark read, mark all read
- Auto-popup on workspace launch (once per session)
- Remove mock NotificationPanel from LegalWorkspace
- Dark mode support

### Out of Scope
- Event-driven triggers (data change detection)
- Date arithmetic in templates
- Custom user-defined notification rules
- Shared/team digests
- Custom notification panel in workspace (use MDA native bell)
