## Overview

Merge the AnalysisWorkspace Code Page and SprkChatPane Code Page into a single unified Code Page. SprkChat becomes an embedded right panel within the Analysis Workspace (React context + callback props) instead of a separate Dataverse side pane communicating via BroadcastChannel. This eliminates cross-pane serialization, duplicate authentication, timing issues, and fixed layout constraints.

## Problem Statement

The Analysis Workspace and SprkChat currently run as two separate Code Pages in separate iframes communicating via BroadcastChannel (SprkChatBridge). This introduces serialization overhead, timing issues, duplicate MSAL authentication, duplicate context resolution, fixed 400px side pane layout, lifecycle complexity, and two separate deployment artifacts. Now that SprkChat is exclusively the Analysis Workspace AI companion, this two-page architecture is overengineered.

## Proposed Solution

Create a unified single-page architecture:
- **AnalysisAiContext** shared React context provides analysis state, editor refs, selection state, and auth to all panels
- **ChatPanel wrapper** mounts SprkChat from `@spaarke/ui-components` with props from context (no BroadcastChannel)
- **Three-panel layout** with Editor (left) + Source Document (center, collapsible) + Chat (right, collapsible) and two draggable splitters
- **Remove SprkChatPane** Code Page, cross-pane infrastructure, and associated Dataverse web resources
- **Single auth** — one MSAL initialization serves both editor and chat

## Scope

### In Scope
- AnalysisAiContext shared React context
- ChatPanel wrapper mounting SprkChat from shared library
- Three-panel layout with two draggable splitters and panel visibility toggles
- Direct editor integration (insert-to-editor, editor selection, SSE streaming via ref)
- Remove SprkChatPane Code Page and all cross-pane infrastructure
- Remove Dataverse artifacts (3 web resources, ribbon button XML, SidePaneManager)
- Unified auth and context resolution
- M365 Copilot handoff support (Chat panel visible on handoff URL)

### Out of Scope
- SprkChat component changes (shared library components unchanged)
- SprkChat extensibility features (separate project: `ai-sprk-chat-extensibility-r1`)
- BFF API changes (same endpoints, different consumer)
- Mobile/responsive layout (Phase 4 polish, not structural)

2. [ ] Zero BroadcastChannel usage — grep returns zero results in AnalysisWorkspace
3. [ ] Single MSAL auth — network tab shows single token request
4. [ ] Insert to editor — AI "Insert" button writes directly to Lexical editor at cursor

7. [ ] Resizable panels — drag splitters to resize all three panels; min widths enforced

9. [ ] Reusability preserved — no AnalysisWorkspace imports in `@spaarke/ui-components`
10. [ ] SprkChatPane removed — no `sprk_SprkChatPane` web resource in deployed solution
11. [ ] Performance — page load time comparable to current (measure before/after)
