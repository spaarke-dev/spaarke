## Portal-gotcha re-wrap verification

Per `.claude/patterns/ui/fluent-v9-portal-gotcha.md`: `Popover` and `Dialog` content render via React Portal and ESCAPE the outer `FluentProvider`. Each portal surface MUST be re-wrapped in its own `<FluentProvider theme={theme}>` for dark mode to propagate.

### Popover — ✅ re-wrapped

`InsightSummaryCard.tsx:624`:
```tsx
<PopoverSurface>
  <FluentProvider theme={portalTheme} className={styles.popoverSurface}>
    {/* ... body renderer ... */}
  </FluentProvider>
</PopoverSurface>
```

### Dialog — ✅ re-wrapped

`InsightSummaryCard.tsx:691`:
```tsx
<DialogSurface>
  <FluentProvider theme={portalTheme}>
    <DialogBody>
      {/* ... */}
    </DialogBody>
  </FluentProvider>
</DialogSurface>
```

`portalTheme` is sourced from props (host supplies the active theme, same instance as the outer wrap). Pattern matches Option A (re-wrap the portal surface) from `fluent-v9-portal-gotcha.md` — the preferred, explicit option.
