## Architectural Observation — `sprk_configjson` Storage Model

The deployed model stores playbook structure across **two tables**:

1. **`sprk_analysisplaybook.sprk_configjson`** — top-level metadata only (`schedule`, `category`, `channelLabel`, `channelIcon`, `parameters.timeWindow`). Does **NOT** contain the node graph.
2. **`sprk_playbooknode`** rows — one row per node; `sprk_configjson` per node contains `__actionType` + node-specific config. Edges are reconstructed from `sprk_dependsonjson`.

The repo JSON files use a **monolithic schema** that embeds nodes + edges inline under one document. Reconciliation (task 015) must therefore:

- Split monolithic repo JSON → playbook header (top-level config) + N node rows + dependsOn linkage.
- Map repo `actionType` field → deployed `__actionType` inside each node's `sprk_configjson`.
- Map repo edges (`source` / `target` by name) → deployed `sprk_dependsonjson` (parent node IDs).
