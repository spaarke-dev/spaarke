You are extracting structured outcome data from a legal document. For
each field, return the extracted value, the verbatim quote from the
document that supports the extraction, and your confidence (0.0-1.0).

If a field is not present in the document or unclear, return null with
confidence 0 and a brief explanation. NEVER fabricate values that the
document does not state. NEVER paraphrase quotes — every quote MUST be
copied verbatim (character-for-character, including punctuation and case)
from the source document. A downstream mechanical verifier will reject
quotes that do not appear in the source.

Schema:
{
  "outcomeCategory": "favorable_to_client | unfavorable_to_client | neutral | mixed | unclear",
  "settlementAmount": <numeric value or null>,
  "settlementCurrency": "<ISO 4217 currency code, default USD>",
  "outcomeDate": "<ISO 8601 date YYYY-MM-DD or null>",
  "matterDurationDays": <integer or null>,
  "keyTerms": [
    {"term": "<short label>", "description": "<one sentence>"}
  ],
  "evidence": {
    "outcomeCategory": "<verbatim quote from document or null>",
    "settlementAmount": "<verbatim quote or null>",
    "outcomeDate": "<verbatim quote or null>",
    "matterDurationDays": "<verbatim quote or null>"
  },
  "confidence": {
    "outcomeCategory": <0.0-1.0>,
    "settlementAmount": <0.0-1.0>,
    "outcomeDate": <0.0-1.0>,
    "matterDurationDays": <0.0-1.0>
  },
  "explanations": {
    "outcomeCategory": "<one-sentence explanation when value is null; omit otherwise>",
    "settlementAmount": "<one-sentence explanation when value is null; omit otherwise>",
    "outcomeDate": "<one-sentence explanation when value is null; omit otherwise>",
    "matterDurationDays": "<one-sentence explanation when value is null; omit otherwise>"
  }
}

Rules:
1. Return JSON only — no preamble, no commentary, no markdown fences.
2. When a field is null, set its evidence to null and confidence to 0.0,
   and add a short entry under "explanations" describing why.
3. When a field is non-null, both its evidence quote AND a confidence
   above 0.0 are REQUIRED. A non-null value with null evidence is
   schema-invalid and will be rejected.
4. Quotes must be verbatim substrings of the document. Do not normalize
   whitespace, do not expand abbreviations, do not translate currency
   symbols.
5. keyTerms is an array (possibly empty) of short labeled terms relevant
   to the outcome — e.g. cure-period clauses, indemnification caps,
   exclusivity grants. Include only terms explicitly stated in the
   document.
6. settlementCurrency defaults to "USD" when an amount is present but
   currency is unstated and the document is US-jurisdiction; otherwise
   return the explicit ISO code.
